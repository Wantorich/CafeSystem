# 커리큘럼 로드맵 — 카페 주문 시스템

단일 도메인을 점진적으로 확장하는 방식으로 진행한다.
요구사항이 추가될수록 초기 설계의 한계가 드러나고, 리팩토링을 통해 OOP 원칙을 체득한다.

---

## Phase 1 — 단일 책임과 캡슐화 (1~2주)

| 세션 | 핵심 개념 | 요구사항 방향 |
|------|----------|-------------|
| 01 | 클래스 책임 분리, getter/setter 지양 | 메뉴 조회 및 기본 주문 생성 |
| 02 | 불변 객체, Value Object | 금액·수량을 원시값 대신 객체로 |
| 03 | 일급 컬렉션 | 주문 항목 목록을 컬렉션 객체로 |

## Phase 2 — 다형성과 인터페이스 설계 (2~3주)

| 세션 | 핵심 개념 | 요구사항 방향 |
|------|----------|-------------|
| 04 | OCP — 조건문을 다형성으로 대체 | 음료 종류 추가 (기존 if-else 폭발 유도) |
| 05 | Strategy 패턴 | 할인 정책 다양화 (멤버십/쿠폰/시간대) |
| 06 | Template Method vs Strategy | 두 접근법 직접 구현 후 트레이드오프 비교 |

## Phase 3 — 의존성 역전과 조합 (2~3주)

| 세션 | 핵심 개념 | 요구사항 방향 |
|------|----------|-------------|
| 07 | DIP — 의존성 주입 수동 구현 | 결제 모듈 교체 가능하도록 역전 |
| 08 | Factory / Abstract Factory | 음료·사이즈·옵션 조합 생성 |
| 09 | Decorator 패턴 | 샷 추가, 시럽 추가 등 옵션 조합 |

## Phase 4 — 실전 설계 (지속)

| 세션 | 핵심 개념 |
|------|----------|
| 10+ | 전체 도메인 리팩토링 (이전 피드백 반영) |
| 심화 | Observer (주문 상태 알림), Composite (세트 메뉴) |
