# 요구사항: 카페 주문 시스템 — 기본 주문 생성

## 배경

소규모 카페의 주문 처리 시스템을 만든다.
손님이 메뉴를 고르고 수량을 지정하면 주문이 생성되고, 최종 금액을 확인할 수 있어야 한다.
지금은 포장/매장 구분, 할인, 옵션(샷 추가 등)은 없다. 가장 단순한 형태의 주문만 다룬다.

---

## 기능 요구사항

- FR-01: 카페는 여러 메뉴를 제공한다. 각 메뉴는 이름과 가격을 가진다.
- FR-02: 손님은 메뉴를 선택하고 수량을 지정하여 주문 항목을 만들 수 있다.
- FR-03: 하나의 주문은 하나 이상의 주문 항목을 포함할 수 있다.
- FR-04: 주문의 총 금액을 계산할 수 있다.
- FR-05: 주문 항목별 소계(단가 × 수량)를 확인할 수 있다.

---

## 제약 조건

- 메뉴 가격은 0원 이하가 될 수 없다.
- 주문 수량은 1개 이상이어야 한다.
- 주문 항목이 없는 주문은 생성될 수 없다.
- 외부 프레임워크(Spring 등) 없이 순수 Java로만 구현한다.
- DB, 파일 저장 불필요 — 메모리에서만 동작해도 된다.

---

## 검증 시나리오

**시나리오 1 — 단일 메뉴 주문**
- 아메리카노(4,500원)를 2잔 주문한다.
- 주문 항목 소계: 9,000원
- 주문 총액: 9,000원

**시나리오 2 — 복수 메뉴 주문**
- 카페라떼(5,000원) 1잔, 치즈케이크(6,500원) 2조각을 주문한다.
- 카페라떼 소계: 5,000원
- 치즈케이크 소계: 13,000원
- 주문 총액: 18,000원

**시나리오 3 — 잘못된 입력 처리**
- 가격이 0원인 메뉴를 생성하려 하면 거부된다.
- 수량 0으로 주문 항목을 만들려 하면 거부된다.
- 주문 항목 없이 주문을 생성하려 하면 거부된다.
