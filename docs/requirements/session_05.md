# 요구사항: 카페 주문 시스템 — 할인 정책

## 배경

카페가 성장하면서 다양한 할인 정책을 도입하기로 했다.
개발팀은 빠르게 구현하기로 했고, 다음과 같은 코드가 작성되었다.

```java
public Price calculateDiscountedPrice(Price originalPrice, String discountType) {
    if (discountType.equals("MEMBERSHIP")) {
        return Price.of((int) (originalPrice.getMoney() * 0.9));
    } else if (discountType.equals("COUPON")) {
        return Price.of(originalPrice.getMoney() - 1000);
    } else if (discountType.equals("HAPPY_HOUR")) {
        return Price.of((int) (originalPrice.getMoney() * 0.8));
    } else {
        return originalPrice;
    }
}
```

이 코드는 현재 잘 동작하지만, 마케팀에서 할인 정책을 매달 바꾸고 새로운 정책을 추가한다.
할인 정책이 추가될 때마다 이 메서드를 찾아 수정해야 하고,
두 가지 할인을 동시에 적용해야 하는 경우에는 코드가 더 복잡해진다.

---

## 기능 요구사항

- FR-01: 멤버십 할인 — 총액의 10% 할인
- FR-02: 쿠폰 할인 — 1,000원 정액 할인
- FR-03: 해피아워 할인 — 총액의 20% 할인
- FR-04: 할인 없음 — 원래 금액 그대로
- FR-05: 주문에 할인 정책을 적용하여 최종 금액을 계산할 수 있다.
- FR-06: 새로운 할인 정책이 추가될 때 기존 코드를 수정하지 않아야 한다.

---

## 제약 조건

- 위의 if-else 구조를 그대로 사용하지 않는다.
- 할인 적용 후 금액이 0원 이하가 될 수 없다. (최소 1원)
- 순수 Java로만 구현한다.

## 검증 시나리오

**시나리오 1 — 멤버십 할인**
- 총액 10,000원에 멤버십 할인 적용 시 최종 금액은 9,000원이다.

**시나리오 2 — 쿠폰 할인**
- 총액 10,000원에 쿠폰 할인 적용 시 최종 금액은 9,000원이다.

**시나리오 3 — 해피아워 할인**
- 총액 10,000원에 해피아워 할인 적용 시 최종 금액은 8,000원이다.

**시나리오 4 — 할인 없음**
- 총액 10,000원에 할인 없음 적용 시 최종 금액은 10,000원이다.

**시나리오 5 — 신규 할인 정책 추가**
- 시즌 할인(15%) 정책을 추가할 때 기존 할인 정책 코드를 수정하지 않아도 된다.
