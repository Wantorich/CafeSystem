# 요구사항: 카페 주문 시스템 — 주문 항목 목록을 객체로

## 배경

현재 `Order`는 주문 항목 목록을 `Collection<OrderItem>`으로 직접 보유하고 있다.
시스템이 커지면서 다음과 같은 문제가 보고되었다.

- 주문 항목 목록에 대한 검증과 연산이 `Order` 곳곳에 흩어져 있다.
- 외부에서 전달받은 컬렉션이 나중에 수정될 경우 주문이 오염될 수 있다.
- "최소 1개 이상의 항목을 포함해야 한다"는 규칙이 `Order`의 생성 로직에 섞여 있다.

---

## 기능 요구사항

- FR-01: 주문 항목 목록은 독립적인 개념으로 표현되어야 한다.
- FR-02: 주문 항목 목록은 생성 후 외부에서 변경될 수 없어야 한다.
- FR-03: 주문 항목 목록은 비어있을 수 없다.
- FR-04: 주문 항목 목록의 총 금액을 계산할 수 있어야 한다.
- FR-05: 기존의 주문 생성, 소계 계산, 총액 계산 기능은 그대로 동작해야 한다.

---

## 제약 조건

- 외부 프레임워크 없이 순수 Java로만 구현한다.
- 기존 검증 시나리오(session_01, session_02)가 여전히 통과해야 한다.

---

## 검증 시나리오

**시나리오 1 — 총 금액 계산**
- 아메리카노(4,500원) 2잔, 치즈케이크(6,500원) 1조각이 담긴 주문 항목 목록의 총 금액은 15,500원이다.

**시나리오 2 — 불변성 보장**
- 주문 항목 목록 생성 후 원본 컬렉션을 수정해도 목록이 변경되지 않는다.

**시나리오 3 — 잘못된 입력 처리**
- 주문 항목이 없는 목록을 생성하려 하면 거부된다.
- null이 전달되면 거부된다.
